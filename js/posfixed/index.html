Gorik: jQuery position fixed

Небольшой скрипт, в виде jQuery плагина, для фиксированного положения элемента на странице во время прокрутки страницы по вертикали.

Подключение:

  • Подключаем библиотеку jQuery 1.8+
  • Подключаем плагин pos.fixed.js
  • Подключаем скрипт для нужного нам элемента:
        $('document').ready(function(){
      $('.search_field').fixeditem();
    });

Параметры:

position - Параметр для вертикального позиционирования элемента на странице. Возможные значения: "top" и "bottom". Значение по умолчанию: "top";

correctY - Значение в пикселях для смещения элемента по вертикали. Для позиционирования "top" - это будет отступ от верхнего края страницы, соответственно для "bottom" - от нижнего. Значение по умолчанию: 0;

fixedClass - CSS класс, который добавляется к элементу после изменения его позиционирования на "fixed". Значение по умолчанию: "state_fixed";

container - Блок, желательно родительский, в пределах которого элемент будет иметь позиционирование "fixed". Возможные значения: ".some_class" и "#some_id". По умолчанию - отсутствует;

rightSide - Параметр, который выравнивает положение элемента, в случае если элемент спозиционирован по правому краю родителя. Для строчных элементов это может быть text-align:right, для блочных - float:right. Возможные значения: true и false. Значение по умолчанию: false;

wrapper - Параметр, который определяет оборачивать или нет элемент в div. Div нужен, что бы не слетала вёрстка, когда меняется позиционирование элемента. Функцию можно отключить, если указать стили для родительского элемента в css. В этом случае у родительского элемента внутренние отступы должны равняться нулю (padding:0;) Возможные значения: true и false. Значение по умолчанию: true;

parentWidth - Параметр, который выставляет ширину элемента после изменения размеров окна браузера. Возможные значения: true и false. Значение по умолчанию: true;
